🧠 Master MVP Prompt — Regex Tester Web App

Build a fast, interactive Regex Tester web app using HTML, CSS, and vanilla JavaScript (no frameworks).

🎯 Core Objective

Create a tool where users can:

Enter a regex pattern
Provide test input text
Instantly see matches highlighted
Understand match groups and results clearly
⚙️ Core Features (MVP)
1. 🔤 Regex Input
Input field for regex pattern
Flags selector:
g (global)
i (case-insensitive)
m (multiline)
s (dotall)
Real-time validation (show error if invalid regex)
2. 📄 Test String Input
Large textarea for input text
Supports multi-line input
Placeholder examples for quick testing
3. 🎯 Match Highlighting (Main Feature)
Highlight all matches in the text
Different colors for:
Full match
Capture groups (optional but recommended)
Show match index positions
4. 📊 Match Results Panel

Display structured results:

Total matches count
Each match:
Matched text
Start & end index
Captured groups (Group 1, 2, etc.)
5. ⚡ Real-Time Execution
Update results as user types
Debounce input (avoid lag on large text)
🚀 Extended Features (to make it stand out)
🧠 Regex Explanation (Very Cool Feature)
Basic breakdown of regex pattern:
\d → digit
. → any character
* → repeat
Show human-readable explanation (even partial is fine)
🎨 Highlight Modes
Toggle:
Highlight all matches
Highlight current match only
Navigate matches:
Next / Previous buttons
🧪 Quick Presets

Buttons to auto-fill examples:

Email regex
URL regex
Phone number regex
Password validation regex
📋 Copy Utilities
Copy matches
Copy regex
Copy results JSON
🧩 Replace Tool (Mini Feature)
Input for replacement string
Show live "replace result"
Button: "Replace All"
📁 Import / Export
Load text from .txt file
Save results as .json
🎯 UI / UX
Split layout:
Left → Inputs
Right → Highlighted output + results
Clean dev-tool style UI (dark mode preferred)
Monospace font for text areas
Sticky results panel
🧱 Technical Requirements
Use:
HTML5
CSS3 (Flexbox/Grid)
Vanilla JavaScript
No backend
No frameworks
Optional: small helper libraries allowed, but prefer native JS
⚠️ Edge Cases to Handle
Invalid regex → show error without breaking app
Empty input → no crashes
Large text → maintain performance
Infinite loops (avoid unsafe regex execution patterns)
